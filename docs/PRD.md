# PRD: 개인 개발 블로그 (Notion CMS)

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 개인 개발 블로그 |
| 작성일 | 2026-02-20 |
| 목적 | Notion을 CMS로 활용한 개인 기술 블로그 |

### 배경 및 목적

Notion에서 글을 작성하면 자동으로 블로그에 반영되는 개인 기술 블로그를 구축한다. 별도의 CMS 관리 화면 없이 Notion 워크스페이스를 콘텐츠 편집 도구로 활용함으로써 글쓰기에만 집중할 수 있는 환경을 만든다.

### CMS로 Notion을 선택한 이유

- 익숙한 Notion 인터페이스에서 글 작성 가능
- 발행/초안 상태 관리가 간편
- 별도 관리자 페이지 구축 불필요
- Notion API를 통해 데이터 실시간 연동

---

## 3. Notion 데이터베이스 구조

| 필드명 | 타입 | 설명 |
|--------|------|------|
| Title | title | 글 제목 |
| Category | select | 카테고리 (예: Frontend, Backend, DevOps 등) |
| Tags | multi_select | 태그 목록 |
| Published | date | 발행일 |
| Status | select | 상태 (`초안` / `발행됨`) |
| Content | page content | 본문 (Notion 페이지 블록) |

> **규칙**: `Status`가 `발행됨`인 글만 블로그에 노출된다.

---

## 4. 주요 기능 요구사항

### 4.1 글 목록 조회

- Notion 데이터베이스에서 `Status = 발행됨` 인 글 목록을 가져온다.
- 발행일(`Published`) 기준 내림차순 정렬로 표시한다.
- 목록에는 제목, 카테고리, 태그, 발행일을 표시한다.

### 4.2 글 상세 페이지

- 개별 글의 Notion 페이지 블록 콘텐츠를 파싱하여 렌더링한다.
- 지원 블록 타입: 단락, 제목(H1~H3), 코드, 이미지, 인용, 목록(순서/비순서), 구분선
- 글 상단에 제목, 카테고리, 태그, 발행일을 표시한다.

### 4.3 카테고리별 필터링

- 카테고리 선택 시 해당 카테고리의 글만 필터링하여 표시한다.
- 전체 카테고리 목록은 Notion 데이터베이스에서 동적으로 가져온다.

### 4.4 검색 기능

- 글 제목 및 태그를 기준으로 클라이언트 사이드 검색을 제공한다.
- 검색어 입력 시 실시간으로 결과를 필터링한다.

### 4.5 반응형 디자인

- 모바일(375px), 태블릿(768px), 데스크탑(1280px) 기준으로 레이아웃을 최적화한다.
- Tailwind CSS의 반응형 유틸리티 클래스를 활용한다.

## 5. 화면 구성

### 5.1 홈 (`/`)

- 최근 발행된 글 목록 표시
- 카테고리 필터 탭
- 검색 입력창

### 5.2 글 상세 (`/posts/[id]`)

- 글 제목, 메타 정보(카테고리, 태그, 발행일)
- Notion 페이지 본문 렌더링
- 이전/다음 글 이동 링크

### 5.3 카테고리 (`/category/[category]`)

- 선택한 카테고리의 글 목록 표시
- 카테고리 헤더 및 글 개수 표시

## 8. 비기능 요구사항

- **성능**: ISR(Incremental Static Regeneration) 또는 SSG를 활용하여 페이지 로딩 속도 최적화
- **보안**: Notion API 키는 서버 사이드에서만 사용, 클라이언트에 노출 금지
- **타입 안전성**: TypeScript strict 모드 적용, `any` 타입 사용 금지
- **접근성**: 시맨틱 HTML 태그 사용, 이미지 alt 속성 필수 적용
